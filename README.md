# API Consolidation Plugin for Claude Code

Automated REST API discovery, validation, and consolidation for any backend project. Scans your codebase, finds every endpoint, runs 9 quality checks, produces a phased fix plan, and applies code changes — all from within Claude Code.

## Supported Frameworks

| Framework | Detection | Fully Supported|
|-----------|-----------|:---------------:|
| Spring Boot (Java) | `pom.xml` / `build.gradle` | Yes |
| Express.js (Node) | `package.json` | Yes |
| FastAPI (Python) | `requirements.txt` / `pyproject.toml` | Yes |
| Django (Python) | `requirements.txt` / `pyproject.toml` | Yes |
| Go (Gin/Mux/Echo) | `go.mod` | Yes |

## Installation

### From GitHub (recommended)

```bash
# Step 1: Add the marketplace
/plugin marketplace add yourusername/api-consolidation-plugin

# Step 2: Install the plugin
/plugin install api-consolidation@api-consolidation-marketplace
```

### From local directory (development)

```bash
claude --plugin-dir ./api-consolidation-plugin
```

## Commands

After installation, these slash commands are available:

| Command | What it does |
|---------|-------------|
| `/api-consolidation:discover-apis` | Scan all controllers, extract endpoints, generate inventory |
| `/api-consolidation:validate-apis` | Run 9 quality checks against all endpoints |
| `/api-consolidation:consolidate-apis` | Produce a phased fix plan with exact code changes |
| `/api-consolidation:fix-apis` | Apply the consolidation plan to source code |
| `/api-consolidation:api-report` | Run full pipeline: discover, validate, plan (one command) |

## Quick Start

### Option A: Full pipeline (recommended for first use)

```
/api-consolidation:api-report
```

This runs discovery, validation, and planning in sequence. Review the generated files in `.claude/docs/`, then:

```
/api-consolidation:fix-apis
```

### Option B: Step by step

```
/api-consolidation:discover-apis          # 1. Build endpoint inventory
/api-consolidation:validate-apis          # 2. Find all issues
/api-consolidation:consolidate-apis       # 3. Generate fix plan
                                          # Review .claude/docs/consolidation-plan.md
/api-consolidation:fix-apis               # 4. Apply fixes
/api-consolidation:validate-apis          # 5. Verify fixes resolved issues
```

## What the 9 Checks Validate

| # | Check | Severity | Catches |
|---|-------|----------|---------|
| 1 | Path Conflict Detection | Critical | Duplicate paths, variable overlap, prefix collision |
| 2 | HTTP Status Code Correctness | High | Wrong status codes (302 for GET, 202 for sync ops) |
| 3 | Response Wrapper Consistency | Medium | Mixed raw objects and ResponseEntity usage |
| 4 | Input Validation Coverage | High | Missing @Valid, no bean validation annotations |
| 5 | Authentication Consistency | Critical | DELETE/PUT without auth, PII without auth |
| 6 | Naming Convention Consistency | Low | Verbs in paths, singular/plural mismatch |
| 7 | Layer Violation Detection | Medium | Repositories in controllers, business logic in controllers |
| 8 | Error Handling Completeness | Medium | Missing global handler, unhandled exceptions |
| 9 | Security Vulnerability Scan | Critical | PCI-DSS violations, mass assignment, PII exposure |

## Generated Files

| File | Generated By | Contains |
|------|-------------|----------|
| `.claude/docs/api-inventory.md` | `discover-apis` | Full endpoint catalog with 12 metadata fields |
| `.claude/docs/api-issues.md` | `validate-apis` | QA report with severity ratings and fix recommendations |
| `.claude/docs/consolidation-plan.md` | `consolidate-apis` | 4-phase fix plan with before/after code blocks |

## Hooks (Automatic)

The plugin installs these hooks:

| When | Trigger | What it does |
|------|---------|-------------|
| Before editing endpoint files | Edit on `*Controller.java`, `*router.ts`, etc. | Reminds to re-validate |
| Before git commit | `git commit` | Warns about unresolved critical issues |
| After editing endpoint files | Edit on endpoint files | Reminds inventory may be stale |
| After agent completes | Any agent finishes | Points to generated docs |
| On session start | Session notification | Shows available commands |

## Configuration

### Permissions

The plugin requests these permissions by default:

- `Read(**)` — read any file (for scanning)
- `Glob(**)` — search for files (for finding controllers)
- `Grep(**)` — search file contents (for finding patterns)
- `Write(.claude/docs/*)` — write generated reports

The `fix-apis` command additionally needs `Edit` permission on your source files, which will be requested when it runs.

### Customizing for your project

No configuration needed. The plugin auto-detects your framework and adapts its scanning patterns, validation checks, and fix recommendations accordingly.
